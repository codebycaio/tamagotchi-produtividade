## 💡 O que é o DevGotchi?

O **DevGotchi** é um Tamagotchi da Produtividade — um pet virtual que reflete diretamente seus hábitos de foco e procrastinação.

Abriu o VS Code? Ele fica feliz. Abriu o TikTok? Ele sofre.

A ideia é simples: **gamificar a produtividade** de um jeito divertido, usando o modelo clássico do bichinho virtual como metáfora para a saúde dos seus hábitos digitais.

> Este projeto nasce como exercício de aprendizado em ADS, mas a visão de longo prazo é evoluí-lo para um **SaaS completo** com skins, planos e integração com sistemas operacionais.

## 🗺️ Roadmap — Visão de SaaS

A ideia é crescer o DevGotchi em fases:

**Fase 1 — MVP (atual)**
- [x] Interface web básica
- [x] Sistema de vida (dano e cura)
- [x] Cadastro personalizado de apps
- [x] Persistência local

**Fase 2 — Produto**
- [ ] Pet flutuante sobre outras janelas (Electron)
- [ ] Integração real com apps abertos no sistema operacional
- [ ] Histórico de produtividade e gráficos
- [ ] Sistema de evolução do pet por streaks

**Fase 3 — Monetização**
- [ ] Skins e temas premium para o pet
- [ ] Plano free vs. Pro
- [ ] Sincronização em nuvem entre dispositivos
- [ ] API pública para integração com ferramentas (Notion, Todoist, etc.)

## 🛠️ Tecnologias

- **HTML5** — Estrutura semântica
- **CSS3** — Dark mode, animações e glassmorphism (sem frameworks)
- **JavaScript (Vanilla)** — Lógica de negócio e manipulação de DOM
- **localStorage** — Persistência de dados no navegador

## 🚀 Como rodar localmente

```bash
# Clone o repositório
git clone https://github.com/seu-usuario/tamagotchi-produtividade.git

# Abra o arquivo no navegador
# Sem necessidade de instalar dependências — é HTML/CSS/JS puro!
open index.html
```

Ou simplesmente arraste o `index.html` para o seu navegador.

## 🧠 Sobre o desenvolvimento

Projeto desenvolvido como parte do meu aprendizado no curso de **Análise e Desenvolvimento de Sistemas (ADS)**.

O processo de desenvolvimento contou com o auxílio do **Antigravity** (ferramenta de pair programming com IA da Google DeepMind) — não para escrever o código por mim, mas como um instrutor interativo: explicando conceitos, revisando lógica e guiando decisões técnicas enquanto eu colocava a mão na massa.
